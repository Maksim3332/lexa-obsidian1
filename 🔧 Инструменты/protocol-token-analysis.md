# Протокол анализа токенов (Token Analysis Protocol)
## v1.0 — 31 May 2026

### Цель
Стандартизированный фреймворк для комплексного анализа мемкоинов и крипто-токенов перед принятием торгового решения.

---

## Этап 1: Идентификация

| Шаг | Действие | Команда/Источник |
|-----|----------|------------------|
| 1.1 | Получить адрес контракта | От пользователя или из поиска |
| 1.2 | Определить сеть | sol / bsc / base / eth |
| 1.3 | Базовый Security Check | `gmgn-cli token security --chain <chain> --address <address>` |
| 1.4 | Информация о токене | `gmgn-cli token info --chain <chain> --address <address>` |

**Критические флаги (abort if ANY):**
- `is_honeypot` = "yes"
- `rug_ratio` > 0.3 (требует re-confirmation)
- `is_mintable` = true
- `is_blacklisted` = true

## Этап 2: Narrative Intelligence (Анализ Мема)

**Ключевое правило:** Сначала разобрать мем, потом привязать к токену. Не судить мем по крипто-твиттеру.

| Шаг | Действие | Инструмент |
|-----|----------|------------|
| 2.1 | Определить происхождение мема | Websearch / TikTok / Instagram |
| 2.2 | Найти оригинальное вирусное видео | websearch / Know Your Mame |
| 2.3 | Проверить Know Your Meme entry | knowyourmeme.com/memes/<name> |
| 2.4 | Проверить освещение в мейнстрим-медиа | websearch: Yahoo News, NYT, BuzzFeed... |
| 2.5 | Определить: органический мем или искусственный? | Дата виральности vs дата создания токена |
| 2.6 | Есть ли культурный вес вне крипты? | YouTube разборы, Instagram репосты, TikTok дуэты |
| 2.7 | Становится ли фраза новым сленгом? | Используют ли люди в реальной жизни / не-крипта твиты |
| 2.8 | **Только после понимания мема** → привязать к токену | Twitter поиск по тикеру + контракту + хендлу |

**Паттерны нарративов:**

| Тип | Пример | Потенциал |
|-----|--------|-----------|
| 🟢 **Вирусный сленг** | "Dat Bih Gah" = новый "fire/gas" | Высокий — культурная экспансия |
| 🟢 **Мейнстрим-мем** | Hawk Tuah, Chill Guy | Средний — цикл мема 2-4 недели |
| 🟡 **Нишевый мем** | Специфичный субкультурный юмор | Низкий — не выйдет за пределы ниши |
| 🔴 **Крипто-созданный мем** | Pump.fun токен без мема | Нулевой — только накачка |
| 🔴 **Фабричный мем** | Создан агентством/бот-фермой | Отрицательный — искусственный |
| 🔴 **Украденный мем** | Мем существовал, токен создан пост-фактум | Низкий — сообщество мема не переходит в крипту |

**Ключевой вопрос:** *"Если бы токен исчез завтра, продолжил бы мем жить?"*
- Да → мем сильный, токен может оседлать волну
- Нет → мем умрёт вместе с токеном

## Этап 4: On-Chain Данные

### 4.0 Важно: контекст данных
**Обрати внимание:** GMGN помечает кошельки как `smart_degen` и `renowned` на основе всей их истории, не только этого токена. Если SM/KOL держат 0.00% — значит они уже вышли. Смотреть на `amount_percentage`, а не просто на наличие тэга.

### 4.1 Пул и Ликвидность
| Метрика | Команда |
|---------|---------|
| Пул токена | `gmgn-cli token pool --chain <chain> --address <address>` |
| Цена, маркеткап, ликвидность | `gmgn-cli token info` |

### 4.2 Холдеры
| Шаг | Действие |
|-----|----------|
| 4.2.1 | Топ-50 холдеров: `gmgn-cli token holders --chain <chain> --address <address> --limit 50` |
| 4.2.2 | **Проверить балансы топ-10 холдеров** (gmgn-portfolio) — сколько у них $, P&L, какие ещё токены держат |
| 4.2.3 | Определить: бандлеры? свежие кошельки? концентрированность? |
| 4.2.4 | SMART MONEY холдеры: `gmgn-cli token holders --chain <chain> --address <address> --limit 50 --tag smart_degen` |
| 4.2.5 | KOL холдеры: `gmgn-cli token holders --chain <chain> --address <address> --limit 50 --tag renowned` |
| 4.2.6 | Трейдеры: `gmgn-cli token traders --chain <chain> --address <address> --limit 20` |

### 4.3 Пул и Метерики
| Метрика | Где смотреть |

## Этап 7: Формирование Вердикта

### Сводная таблица
```
Токен:        $TICKER
Адрес:        <contract>
Сеть:         <chain>
Цена:         $X.XX
ATH:          $X.XX (-XX% от ATH)
Маркеткап:    $X.XXM
Ликвидность:  $X.XXK
Холдеров:     X,XXX
24h объём:    $X.XXM
CEX листинги: <list>
```

### Факторы решения
| Фактор | 🟢 Бычий | 🔴 Медвежий |
|--------|---------|------------|
| Продукт | Работает, есть utility | Только мем, ничего нет |
| Команда | Верифицирована, известна | Анонимна, скам-история |
| Smart Money | Заходят/держат | Все вышли |
| Snipers | Частично держат | Все продали |
| Ликвидность | >$100K, залочена | <$10K |
| Social | Активное комьюнити | Боты, тишина |
| Листинги | CEX листинги | Только DEX |
| Объём | Растёт | Падает |

### Итоговый вердикт
- **СИЛЬНАЯ ПОКУПКА** — все факторы зелёные
- **ПОКУПКА** — продукт реален, но цена падает
- **ДЕРЖАТЬ** — есть потенциал, ждать катализатора
- **ПРОДАВАТЬ** — фундамент сломался
- **ИЗБЕГАТЬ** — красные флаги

## Памятка: Проверка балансов холдеров (Holder Wallet Check)

Для топ-10 холдеров (или подозрительных) выполнять:
```bash
gmgn-cli portfolio --chain <chain> --address <wallet_address>
```

**Что смотреть:**
- Общий P&L (realized + unrealized)
- Win rate
- Сколько ещё токенов держит
- Какие ещё токены в портфеле (диверсификация?)
- Похож на реального трейдера или бандлерский кошелёк?

**Признаки бандлера:**
- Создан за 5 мин до запуска токена
- Только buys, никогда не sells
- Идентичные суммы/тайминги с другими "холдерами"
- Пустой P&L по другим токенам

## Checkpoint (сквозной)

Перед началом анализа — проверить checkpoint:
```bash
checkpoint latest <address>
```
Если вернул `{"stage": N}` с N > 0 — пропустить выполненные этапы, загрузить данные через `checkpoint load`.

После завершения каждого этапа — сохранить:
```bash
checkpoint save <address> --stage <N> --data '<собранные_данные_json>'
```

После финального вердикта — пометить завершённым:
```bash
checkpoint done <address>
```
