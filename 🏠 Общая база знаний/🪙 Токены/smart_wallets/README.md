# Smart Wallets Database

Отслеживание кошельков, которые сидели в успешных токенах (выросли после алерта).

## Структура

| Файл | Описание |
|------|----------|
| `smart_wallets.json` | Мастер-база: все кошельки с токенами, профитами, тэгами |
| `README.md` | Этот файл — человекочитаемый отчёт |

## Как пополняется

1. Каждый день `token-post-analysis.py` проверяет 30 токенов
2. Если токен вырос на 5%+ с прошлой проверки — извлекаются топ-20 Smart Money трейдеров
3. Кошельки записываются в `smart_wallets.json`
4. Если кошелёк уже есть — обновляется его профиль (добавляется токен, растёт счётчик)

## Формат записи

```json
{
  "address": "wallet_addr",
  "name": "display_name_or_short",
  "tags": ["smart_degen"],
  "first_seen": 1716800000,
  "last_seen": 1716886400,
  "tokens": [
    {
      "addr": "token_addr",
      "symbol": "TOKEN",
      "mc_at_entry": 100000,
      "profit": 5000,
      "realized_pnl": 0.5
    }
  ],
  "total_tokens": 1,
  "total_profit": 5000
}
```

## Использование

- **Поиск общих кошельков** — если один кошелёк сидел в 3+ успешных токенах, это сильный сигнал
- **Copy-trading** — отслеживать новые входы этих кошельков через GMGN
- **Улучшение алертов** — если в токене есть хотя бы 1 кошелёк из этой БД, повышать приоритет
